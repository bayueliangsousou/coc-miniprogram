# 技能点数系统 - 实现总结

## ✅ 已实现功能

### 1. 技能点数计算逻辑 (utils/character.js)

**CoC 7th Edition 规则：**
- **总点数 = INT × 2**
- **兴趣技能最多可用 10 点**（用于职业技能外的技能）
- **点数消耗 = 当前值 - 基础值**

**calcSkillPoints() 函数：**
```javascript
{
  total: INT * 2,           // 总可用点数
  used: 已消耗点数,         // 已使用点数
  remaining: 剩余点数,      // 剩余可用点数
  occUsed: 职业技能已用,    // 职业技能消耗
  intUsed: 兴趣技能已用      // 兴趣技能消耗
}
```

## 🎯 交互流程

### 创建角色流程：
```
1. 填写基本信息
   ↓
2. 骰点设置属性（INT 决定技能点数）
   ↓
3. 选择职业
   ↓
4. 编辑技能（看到可用点数）
   ↓
5. 分配技能点数
   - 职业技能：从总点数中分配
   - 兴趣技能：最多用 10 点
   ↓
6. 实时查看剩余点数
   - 超支时红色警告
   - 保存前验证
   ↓
7. 保存角色
```

### 编辑已有角色流程：
```
1. 进入技能编辑页
   ↓
2. 查看当前点数使用情况
   ↓
3. 调整技能值
   - 可以减少已用点数
   - 重新分配点数
   ↓
4. 保存验证
   - 确保点数不超支
   ↓
5. 保存修改
```

## 📊 点数计算示例

### 示例 1：新角色
- INT = 70
- 总点数 = 140 点
- 分配：
  - 侦查（职业技能）：50 点（基础 25 + 投入 25）
  - 图书馆使用（职业技能）：60 点（基础 20 + 投入 40）
  - 闪避（基础）：35 点（DEX 70 ÷ 2）
  - 驾驶（兴趣）：50 点（基础 20 + 投入 30）
- 使用点数：25 + 40 + 30 = 95 点
- 剩余点数：45 点

### 示例 2：点数超支
- INT = 50
- 总点数 = 100 点
- 已用：120 点
- 剩余：-20 点（红色警告）
- 无法保存

## 🔧 技术实现细节

### 数据流
```

  ↓ 计算

  ↓ 更新

  ↓ 渲染

```

### 性能优化
- 使用 `setData` 的回调确保数据同步
- 每次技能输入后重新计算点数
- 避免频繁的全量数据更新

## 📝 规则说明

### CoC 7th Edition 技能点数规则
1. **职业点数**：由职业技能获得，可无限投入
2. **兴趣点数**：最多 10 点，用于非职业技能
3. **基础值**：不消耗点数
4. **点数投入**：1 点 = 技能值 +1

### 点数计算公式
```
总点数 = INT × 2

已用点数 = Σ (技能当前值 - 技能基础值)

剩余点数 = 总点数 - 已用点数
```
